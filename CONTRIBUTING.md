# Contributing

Contributions are welcome when they improve research integrity, HCI-method fit,
cross-platform behavior, or reproducibility.

Before submitting a change:

1. Keep personal information and real participant data out of fixtures.
2. Add or update a focused test for behavioral changes.
3. Run `bash verify.sh`.
4. Explain the research or workflow failure being fixed.
5. Keep `SKILL.md` concise; move detailed, conditional guidance into references.

Do not add venue requirements from memory. Link or encode current official
requirements only when they can be maintained and verified.
